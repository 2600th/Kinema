# Lessons

- Keep `package-lock.json` generated by npm; avoid manually editing platform-specific optional package entries (for example Biome CLI variants), because missing `version` metadata can break CI installs with `Invalid Version`.
- For iPhone/iPad browser issues, prefer `visualViewport`-driven resize handling plus `orientationchange` follow-up resizing, and default Apple mobile browsers to the compatibility WebGL renderer path unless there is a verified reason to use the experimental renderer.
- When Apple mobile browsers are on the compatibility WebGL renderer path, gate all `three/webgpu` or TSL-dependent scene content behind the same capability check and provide a plain Three.js fallback, or procedural loads can fail partway through even when the base world renders.
- Shared menu/modal shells should use viewport-based max-heights plus scroll fallback on short screens; a centered glass card with `overflow: hidden` can silently clip reachable actions in mobile landscape even when the card itself still fits on screen.
- Treat audio as optional on mobile/WebKit bootstrap: if Tone.js node construction fails, degrade to a silent no-op controller instead of aborting the whole app, or graphics/load bugs can be misdiagnosed as procedural-level failures.
